```yaml
Программист — передать в функцию произвольное количество аргументов
    Решение: Создать функцию с использованием аргументов *args и/или **kwargs
```
```py
def adder(*nums):
    sum = 0

    for n in nums:
        sum += n

    print("Sum: ", sum)

adder(3, 5)
adder(4, 5, 6, 7)
adder(1, 2, 3, 5, 6)

```
```py
def intro(**data):
    print("\nData type of argument: ",type(data))

    for key, value in data.items():
        print("{} is {}".format(key, value))

intro(Firstname="Sita", Lastname="Sharma", Age=22, Phone=1234567890)
intro(Firstname="John", Lastname="Wood", Email="johnwood@nomail.com", Country="Wakanda", Age=25, Phone=9876543210)
```
```yaml
    Другие способы решить проблему:
        - Передать в функцию именованный аргумент списком
    Значимые преимущества/недостатки:
        - (+) Можно передавать любое произвольное количество аргументов
        - (+) Можно указывать аргументы либо по позиции, либо по ключевому слову
        - (-) Все слоты обычных аргументов должны быть заполнены до того, как будет заполнен слот с переменным аргументом
        - (-) Если существует параметр с тем же именем, что и ключевое слово, то значение аргумента присваивается этому слоту параметра. Однако если слот параметра уже заполнен, это ошибка
    Причины преимуществ/недостатков:
        - С помощью *args можно создать более гибкий код, который принимает различное количество аргументов без ключевых слов в функции.
        - Использование **kwargs дает гибкость в использовании ключевых аргументов в программе. Когда мы используем **kwargsв качестве параметра, не нужно знать, сколько аргументов мы в конечном итоге хотели бы передать функции.
        - Использование *args и **kwargs в первую очередь для обеспечения удобочитаемости и удобства
```


```yaml
Программист — обеспечить выполнение функции при отсутствии некоторых аргументов
    Решение: сделать аргумент с дефолтным значением
```
```py
def func(x, y=1):
    return x/y
```
```yaml
    Другие способы решить проблему:
        - Передать в функцию глобальную переменную
    Значимые преимущества/недостатки:
        - (+) Все переменные уложены внутри функции
        - (+) Отсутствие ошибок при вызове функции в другом скрипте или при отсутствии аргумента
        - (-) Значение по умолчанию оценивается только один раз при вызове функции
    Причины преимуществ/недостатков:
        - Значения по умолчанию указывают, что аргумент функции примет это значение, если во время вызова функции не будет передано значение аргумента.
        - Значения по умолчанию сокращают возможность ошибок при вызове функции.
```
